## API Categories

The Adamik API is organized into four main categories:

- **Chain**: Query blockchain network information and validators
- **Address**: Handle address encoding and validation
- **Account**: Retrieve account states, balances, and history
- **Transaction**: Manage transaction lifecycle from encoding to broadcasting

```mermaid
sequenceDiagram
    participant CLIENT as Client App
    participant ADAMIK as Adamik API

    %% Chain & Network Info %%
    Note right of CLIENT: Chain Information
    CLIENT->>ADAMIK: Query chain details/validators
    ADAMIK->>CLIENT: Return chain data

    %% Account Operations %%
    Note right of CLIENT: Account Operations
    CLIENT->>ADAMIK: Get account state/tokens/history
    ADAMIK->>CLIENT: Return account information

    %% Transaction Lifecycle %%
    Note right of CLIENT: Transaction Lifecycle
    CLIENT->>ADAMIK: Encode transaction
    ADAMIK->>CLIENT: Return encoded transaction
    CLIENT->>ADAMIK: Broadcast transaction
    ADAMIK->>CLIENT: Return transaction result
```
