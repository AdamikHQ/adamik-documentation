The Adamik API adheres to RPC principles, providing streamlined and predictable interactions for blockchain integration in cryptocurrency applications. It features resource-oriented URLs, accepts and returns JSON-encoded data, and uses standard HTTP methods and response codes.

## Focused Functionality

Each API request processes single objects to ensure precise updates, supporting clear and straightforward blockchain interactions.

## OpenAPI specification

The Adamik API is formalized in an OpenAPI Description (OAD).
You can retrieve the complete specification by using the [OpenAPI endpoints](endpoint/get-openapi).

## Versioning

Currently in beta, the Adamik API is continuously enhanced to meet diverse needs with new versions and features.
